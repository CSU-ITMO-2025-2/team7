база данных postgres.
таблица 1: users: id, login, password_hash, created_at
таблица 2: datasets: id, user_id, s3_path, name, created_at
таблица 3: runs: id, user_id, dataset_id, status, configuration, created_at

создать пользователя:
post ручка
1. принимает логин и пароль
2. хэширует пароль используя passlib
3. создает запись в таблице users

создать датасет:
post ручка
1. принимает csv файл, название датасета, валидирует его:
    - проверяет, что файл не пустой
    - проверяет что валидный csv
    - проверяет, что в файле есть заголовки
    - проверяет что в файле не более 10000 строк и не более 50 столбцов
2. из jwt получает user_id
3. загружает файл в s3 (используя aiobotocore) по пути s3://datasets/{user_id}/{dataset_name}/{dataset_name}.csv
4. создает запись в таблице datasets с путем s3

создать run:
post ручка
1. принимает id датасета, конфигурацию в json
2. из jwt получает user_id
3. из pg получает s3 путь датасета
4. создает запись в таблице runs со статусом "in queue" и конфигурацией
5. пишет в кафку сообщение с user_id, dataset_s3_path, run_id и конфигурацией

получить список датасетов пользователя:
get ручка

получить список запусков пользователя:
get ручка

изменить статус запуска:
post ручка
1. принимает id запуска и новый статус (один из: "in queue", "processing", "completed", "failed") нужно сделать для этого enum
