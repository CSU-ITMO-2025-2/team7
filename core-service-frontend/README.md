# Core Service Frontend

Простой статичный UI на HTML/JS для работы с `core-service` без правок бэкенда. Поддерживает:
- регистрацию и получение JWT;
- загрузку CSV-датасетов и просмотр списка;
- создание запуска по выбранному датасету с JSON-конфигурацией;
- отдельную страницу с историей запусков.

## Быстрый старт
1. Запустите `core-service` на `http://localhost:8000`.
2. Поднимите статический сервер для фронтенда:
   ```bash
   cd core-service-frontend
   python -m http.server 5500
   ```
3. Откройте `http://localhost:5500/index.html` (создание датасетов/запусков) или `http://localhost:5500/runs.html` (просмотр запусков).

JWT сохраняется в `localStorage` под ключом `coreServiceToken`, поэтому после логина можно обновлять страницу без повторного ввода.

## CORS
`core-service` сейчас не отдаёт CORS-заголовки. Если открываете фронт с другого порта, браузер может блокировать запросы. Варианты решения:
- повесить фронт за обратным прокси, который направляет `/` на статические файлы, а `/auth`, `/datasets`, `/runs` — на `http://localhost:8000`;
- добавить `CORSMiddleware` в сервис (если это допустимо для вас);
- использовать расширение/флаг браузера для локальной разработки.
